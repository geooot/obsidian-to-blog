# obsidian-to-blog

A CLI tool that uses the Obsidian CLI (v1.12.4+) to generate a static HTML site from an Obsidian vault. Given an entrypoint markdown file, it generates HTML pages for the file and all files linked from it (traversing the link graph), including links generated by plugins like Dataview.

## Core Concept

The Obsidian CLI operates as a "remote control" for a running Obsidian instance. This tool leverages multiple CLI commands to:

1. Open files in Obsidian with `obsidian open`
2. Switch to preview mode with `obsidian command id="markdown:toggle-preview"`
3. Wait for plugins (Dataview, etc.) to render (Node.js setTimeout)
4. Extract the fully-rendered DOM HTML with `obsidian dev:dom`
5. Parse links from the rendered output (catching plugin-generated links)
6. Generate standalone HTML pages

### Important: Async Race Condition Workaround

The Obsidian CLI has a known race condition where async operations inside `obsidian eval` may not return results properly. When using `await` or `setTimeout` inside eval code, the CLI process may exit before stdout is flushed.

**Our solution:** Instead of using a single async `eval` script, we use separate synchronous CLI commands:
- `obsidian open` - Opens a file (synchronous)
- `obsidian command` - Toggles preview mode (synchronous)
- Node.js `setTimeout` - Handles the wait for plugins to render
- `obsidian dev:dom` - Extracts rendered HTML (synchronous)

This avoids the race condition entirely.

## Key Components

### 1. Obsidian CLI Wrapper (`obsidian.ts`)

Wraps the `obsidian` command-line tool:

```typescript
class ObsidianClient {
  // Check CLI availability and Obsidian is running
  checkConnection(): Promise<boolean>;
  
  // Read file content
  read(file: string): Promise<string>;
  
  // List files in vault
  listFiles(folder?: string): Promise<string[]>;
  
  // Execute JavaScript in Obsidian's runtime (synchronous only!)
  eval(code: string): Promise<string>;
  
  // Open a file in Obsidian
  openFile(file: string): Promise<void>;
  
  // Run an Obsidian command by ID
  runCommand(commandId: string): Promise<void>;
  
  // Get DOM element content using dev:dom
  getDom(selector: string, inner?: boolean): Promise<string>;
  
  // Get the configured attachments folder path
  getAttachmentsFolder(): Promise<string>;
  
  // Resolve a link path using Obsidian's resolver
  resolveLinkPath(linkPath: string, sourcePath: string): Promise<string | null>;
}
```

Uses multiple CLI commands to extract rendered HTML:

```typescript
    // Step 5: Parse links and images from HTML using cheerio
    // ...
  }

```

### 3. Link Crawler (`crawler.ts`)

BFS traversal of the link graph:

```typescript
class LinkCrawler {
  private visited: Set<string>;
  private queue: string[];
  
  // Crawl from entrypoint, discovering all linked files
  async crawl(entrypoint: string): Promise<Map<string, ExtractedContent>>;

```

Two-pass link discovery:
1. **Pass 1**: Extract rendered HTML and parse `<a href>` / `data-href` attributes (catches plugin-generated links like Dataview tables)
2. **Pass 2**: Parse raw markdown for `[[wikilinks]]` and `[text](path)` links as backup

Generates standalone HTML pages:

## Technical Details

### HTML Extraction (Multi-Command Approach)

Instead of a single async eval script, we use separate commands:

```bash
# 1. Open the file
obsidian open vault="MyVault" file="Note Name"

# 2. Switch to preview mode
obsidian command vault="MyVault" id="markdown:toggle-preview"

# 3. Wait in Node.js (not in Obsidian)
await sleep(500);

# 4. Extract the rendered HTML
obsidian dev:dom vault="MyVault" selector=".markdown-preview-view" inner
```

This approach:
- Avoids the async race condition in `obsidian eval`
- Captures fully-rendered content including Dataview tables
- Works reliably with all plugins

### Image Handling

Images in Obsidian can be referenced as:
- `![[image.png]]` - Wikilink style
- `![alt](image.png)` - Markdown style
- `app://hash/path/to/image.png?timestamp` - Internal Obsidian URLs

The generator:
1. Queries the vault's configured attachments folder via `obsidian eval`
2. Finds images in the rendered HTML (`<img src="...">` and `<span src="...">`)
3. Resolves image paths (handles `app://` URLs, relative paths, attachments folder)
4. Copies images to `{output}/assets/`
5. Rewrites `src` attributes to point to copied files

### Link Rewriting

Internal links are converted from Obsidian format to HTML paths:
- `[[Note Name]]` ‚Üí `note-name.html`
- `[[Folder/Note]]` ‚Üí `folder/note.html`
- `data-href="Note Name"` ‚Üí `href="note-name.html"`
- Links to uncrawled files are marked with `class="broken-link"`

### Vault Name Parsing

The `obsidian vault` command returns tab-separated key-value pairs:
```
name    MyVault
path    /home/user/Documents/MyVault
files   333
folders 4
size    25381070
```

We parse the `name` field to use in subsequent commands with `vault="MyVault"`.

## Error Handling

- **Obsidian not running**: Error with instructions to start Obsidian
- **CLI not found**: Error with setup instructions (ENOENT)
- **File not found**: Warning, skip file, continue with others
- **Plugin timeout**: Extract whatever is rendered after wait time
- **Image not found**: Warning, search common locations (attachments/, assets/, images/)
- **Broken links**: Mark with CSS class, don't fail

## Known Limitations

1. **Obsidian must be running**: The CLI is a "remote control", not a headless tool
2. **Desktop only**: No CLI on mobile (iOS/Android)
3. **Sequential processing**: Each file requires multiple CLI calls, which is slower than batch processing
4. **Preview mode toggle**: Each file extraction toggles preview mode, which may cause visual flicker
